This node lets you send the job to another workflow. All job related data are trasmitted to the defined workflow. You can also specify the recieving node in the workflow.

ID

Defines the workflow ID on which to send to. When the config window is open you can select workflow from the workflow list in the manager and the config will automatically update. You can also use variables holding a proper workflow ID.

Node ID

Defines the starting node in the defined workflow. By default the first found start node is selected. Also supports variables holding a proper node ID

Inherit farm settings

When this option is set the workflow recieving the job will use the parent workflow farming settings even if it has its own farming setting.

Inherit workflow name

Normally the current workflow name will be used when displayed in status monitors. With this option enabled the defined workflow will use the parent workflow name.
