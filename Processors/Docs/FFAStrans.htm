Welcome to the FFAStrans Workflow Manager. From here you can configure FFAStrans and create workflows.

Main menu

File

Import workflow(s)

Imports valid workflows packs (. json) into FFAStrans. It also supports importing legacy (. xml) files.

Export workflow(s)

Exports the selected workflow as an .ini file. If a folder i selected, all the workflows within that folder are exported.

Export all workflows

Exports all the workflows in FFAStrans.

Shut down FFAStrans farm

Shuts down and exits all instances of FFAStrans in the farm. In order to give some farm protection you need admin credential to perform this operation.

Exit FFAStrans

Exits the local instance of the FFAStrans Workflow Manager. Note that this will not stop a running FFAStans service.

Workflows

Search all workflows...

Opens a free text search window. Also works with Ctrl+F

Start all enabled workflows

Starts all enabled workflows..

Stop all running workflows

Stops all running workflows.

Enable all disabled workflows

Enables all disabled workflows.

Disable all enabled workflows

Disables all enabled workflows.

Options

Show status monitor

Opens the FFAStrans monitor where you can monitor and abort jobs being executed by workflows. If the FFAStrans folder is on a shared path, the monitor can be opened by any computer with access to the shared path. The status monitor can also be run without starting the main GUI by running FFAStrans.exe with "-monitor" as argument.

Configuration

Opens the global configuration for FFAStrans.

User variables

Opens the user variables windows to manage user variables and statics.

Clear all job work cache

Clear all cache files from work folders. NOT available in passive mode!

Warning: Make sure NO jobs are running when performing this operation.

Workflows window

The pane title will always show the name "Workflows" and the name of the environment if set in the main configuration. Right clicking this title gives you the option to collapse or expand folder views.

If you right click you mouse on top of a workflow you will get the following context menu:

Start:

Start the selected workflow. If a folder is selected, FFAStrans will start all workflows in that folder.

Stop:

Stop selected workflow. If a folder is selected, FFAStrans will stop all workflows in that folder.

Enable:

This enables disabled workflow. If a folder is selected, FFAStrans will enable all disabled workflows in that folder.

Disable:

This disables the workflow. Disabled workflows can not be started unless they are enabled. If a folder is selected, FFAStrans will disable all enabled workflows in that folder. Disabled workflows does not accept submitted jobs, neither manually nor through the API.

Submit file(s):

This allows you to submit one or multiple files directly to the workflow. This will work whether the workflow has started or not.

Submit file from input:

Opens a simple input window where you can manually insert a complete path to a file for submitting. Note that for the "YouTube decoder" processor a complete URL is also supported.

Reset:

Resets all of the workflows caches. Note that this will also reset all of the workflows monitors and thus make them reprocess all the files monitored.

New folder:

Creates a new folder. You can not create a folder within a folder.

New workflow:

Creates a new workflow with default name 'untitled'.

Rename:

Rename the selected workflow or folder

Delete:

Deletes the selected workflow or folder.

Duplicate:

Creates a copy of the selected workflow.

Move to workflow:

Enables you to move the workflow to a different folder or tor oot.             

Archive. . .:

Retrieve archived versions of the workflow. Note that you can always return to the latest.               

Properties:

Opens the properties and configuration of the workflow.

Copy ID to clipboard:

Copies the unique workflow ID to the clipboard.

Workflow editor

The pane title will always show the name of the workflow. Right clicking this title gives you the option to show parent workflow and description if set. Double clicking the title pane opens up the description field for the workflow.

If you right click you mouse inside the editor you will get the following menu:

Insert processor:

Chose the processors of your desire on which you build your workflow.

Insert workflow:

Adds a node to represent another workflow. This option enables forwarding a job to another workflow which then will act as if it's a part of the parent workflow.

Auto arrange:

This will arrange your processor nodes in a tidy fashion with left-right orientation.

Note: The algorithm might give some strange result on more complex workflows.

Enable snapping:

This enables snapping of nodes to the grid. This works even if the grid does not show.

Show grid:

Shows the grid on which processor nodes snap.

Search nodes...:

Opens a free text search window. The node matching the search criteria will be highlighted as if selected.

Select all:

Selects all the processor nodes for copying.

Select none:

De-selects all the processor nodes.

Copy processor(s):

Copies the selected nodes into node clip board

Paste processor(s):

Copies the selected nodes from node clip board. The processors will be placed at the bottom of you workflow.

Delete processor(s):

Deletes the processor. You can also press del when the processor is selected.

Save workflow:

If you make a modification to your workflow this option will be enabled. Note that there are two kinds of modifications you can do to a workflow. If you change settings of a node then the pane editor title will light up red and show an asterisk (*) next to the workflow name.

Note that if you simply move or rearrange the processor nodes, you can still save the workflow, but there will be no asterisk (*) next to the workflow name in the editor pane title and it will not light up red.

Processor nodes

General configuration: (Double click in the node)

Name:

The default or custom name of the node.

Node ID:

The unique identifier of the node.

Description:

Custom discription of the node. This is used defined and has no default.

Job processing slots:

Defines how many slots the node will occupy. E.g. one might want to set encoder nodes to occupy more slots than lighter nodes in order to prevent overload.

Hosts group:

Sets what hosts group the node can run on. This will make sure that the node only runs on the hosts defined in the group. Defining groups is done in the man general FFAStrans configuration.

Main nodes window: (Rightclick)

Submit file(s). . .

Chose the processors of your desire on which you build your workflow.

Submit file from input. . .

Opens a simple input window where you can manually insert a complete path to a file for submitting.

Copy processor:

Copies the selected node into node clip board

Delete processor:

Deletes the selected node from workflow

Bypass processor:

Disables the selected node and thus preventing it from running.

Connect to selected:

Connects the output of the current node to all selected nodes.

Change processor to. . . :

Lets you choose a new processor node to replace the current. It will keep all connections but you will loose settings from the previous processor node.

Copy ID to clipboard:

Copies the unique node ID for the selected workflow to the clipboard.

Node input connector: (Right click)

Execute on success

Executes the processor if the previous completed with success.

Execute on error

Executes the processor if the previous completed with an error message.

Execute on both

Executes the processor regardless of the previous processor completed state.

Node output main connector: (Left click + drag)

In order to connect processor you must click mouse on the output-connector of a processor and drag the cursor to the destination processor.

Node output sub connector: (Left click + drag + SHIFT)

This method will create a sub connection (brown) to a processor. Sub connectors will be executed before main connectors (black).

To open and configure a processor just click on the processor title name on the processor node.

Short cut keys and mouse operation:

SHIFT + Left click (hold)

Enables multi node selection.

ALT + Left click (hold)

Enables lasso for selecting multiple nodes.

CTRL + Left click (hold)

Enables deleting both nodes and connections.

About "Passive mode"

If you execute the FFAStrans. exe binary with the "-passive" argument, the FFAStrans manager will start as normal but the computer from which you started it will NOT act as a working host. This is ideal if you just want to manage a FFAStrans environment from another computer on the network.

In passive mode the the "Workflow" title pane will have a pear-ish color and the title will display "FFAStrans  Workflow Manager (-PASSIVE MODE-)".
